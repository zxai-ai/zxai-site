# Dispatch -- Lane 3: Report Page

## Build
HTML page that renders a branded one-page report from D1 data. No PDF generation. Customer can print-to-PDF if they want a file.

## Read First
- `shared/types.ts` -- ReportData structure
- `shared/report-schema.json`

## Files to Create

### `src/pages/report.html`
Single HTML file with inline JS. Reads order_id from URL path, fetches report data from API, renders.

Layout (one page, dense, high-signal):
- **Header**: ZxAI logo, practice name, audit date
- **Overall Score**: Big number (0-100) with color coding (red <40, yellow 40-70, green >70)
- **5 Dimension Scores**: Horizontal bar or simple number cards. SEO, GEO/AI, Local, Content, Competitor Gap
- **Top Findings**: 3-5 bullet points, each with the dimension it relates to and why it matters
- **Recommendations**: Top 3, ranked. Each has: what to do, expected impact, difficulty tag
- **CTA Block**: "Want us to fix this?" + booking link + one sentence about the retainer service
- **Footer**: "Generated by ZxAI" + print button

Style: Same palette (navy, white, green). Professional. Prints cleanly (use `@media print` to hide nav elements and print button).

### Report API route in `src/workers/pipeline.ts` (or a small addition)
- `GET /api/report/{order_id}` -- returns report JSON from D1
- No auth needed (URL is the access token -- order IDs are UUIDs, not guessable)

## Done When
- `/report/{test_order_id}` renders correctly with all sections
- Scores display with correct color coding
- Print-to-PDF produces a clean one-page document
- CTA booking link works
- Page loads in under 2 seconds
- No em dashes anywhere in rendered content

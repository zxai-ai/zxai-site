# Dispatch Prompt -- Lane 4: Report Generator + Delivery

## What You Are Building
A Cloudflare Worker that takes the structured report JSON, generates a branded PDF, uploads it to R2, and sends it to the customer via Resend. This is the delivery layer.

## Read First
- `shared/types.ts` -- `ReportData`, `Env` types
- `shared/report-schema.json` -- the input JSON structure
- `shared/brand/` -- brand assets (if they exist; otherwise use ZxAI defaults: navy #1a1a2e, white, accent green #4ade80)

## What to Build

### 1. Report Worker (`src/workers/report-generator.ts`)
Triggered when order status changes to `generating`.

Flow:
1. Read order + report JSON from D1
2. Generate PDF from report data
3. Upload PDF to R2 bucket
4. Send email with PDF link via Resend
5. Update order status to `delivered`
6. Store PDF URL in reports table

### 2. PDF Generator (`src/lib/pdf-generator.ts`)
Generates a branded PDF from ReportData.

Use a lightweight approach that works in Workers (no heavy dependencies):
- Option A: Build HTML report, use a headless rendering service
- Option B: Use `@react-pdf/renderer` if it runs in Workers
- Option C: Build PDF manually with `pdf-lib`

Pick whichever works in Cloudflare Workers runtime. The PDF should include:
- Cover page: ZxAI logo, practice name, audit date, overall score (big number)
- Score overview: visual scorecards for all 5 dimensions
- Section pages: one per dimension with findings and impact
- Recommendations page: prioritized action list
- CTA page: "Want us to fix this for you?" with booking link
- Footer on every page: "Generated by ZxAI" + date

Design: Clean, professional. Navy header bars. White background. Scores use color coding (red/yellow/green). No clutter.

### 3. Email Sender (`src/lib/email.ts`)
Sends the report via Resend.
- `sendReport(email: string, pdfUrl: string, practiceName: string, overallScore: number, apiKey: string): Promise<void>`
- From: `reports@zxai.ai` (or whatever is verified in Resend)
- Subject: "Your AI Visibility Audit Report is Ready"
- Body: Short, plain text. 3 sentences max. Link to PDF. No em dashes.
- Include overall score in the email body as a hook

### Error Handling
- If PDF generation fails: retry once, then set status to `failed`
- If email send fails: retry twice (Resend is reliable), then set status to `failed` but keep PDF URL (manual recovery possible)

## Validation
- [ ] Given test ReportData JSON, generates a valid PDF
- [ ] PDF has all sections: cover, scores, findings, recommendations, CTA
- [ ] PDF uploads to R2 and URL is accessible
- [ ] Email sends via Resend with correct subject and PDF link
- [ ] Order status updates to `delivered`
- [ ] PDF URL stored in reports table
- [ ] Email body contains no em dashes, is under 65 words
